LinkedIn Art Feed

Privacy Policy — Last updated March 7, 2026

Data Collection

None

LinkedIn Art Feed does not collect, track, or transmit any personal data. There are no analytics, no telemetry, and no advertising.

Local Storage

The following is stored on your device using Chrome's storage API. This data never leaves your browser:

  • Preferences — Feed toggle state, selected museum source, and text-to-speech voice preference.
  • Artwork cache — Recently fetched artwork metadata (titles, artists, image URLs) to speed up loading.
  • Gemini API key — Stored locally if you choose to use the AI explanation feature. Never shared with us or any third party.

External Services

Metropolitan Museum of Art API
collectionapi.metmuseum.org
Fetches public domain artwork data. No user data is sent.
Art Institute of Chicago API
api.artic.edu
Fetches public domain artwork data. No user data is sent.
Google Gemini API (optional)
generativelanguage.googleapis.com
Only used when you provide your own API key and click “Explain” on an artwork. Sends artwork metadata (title, artist, date, medium) to generate an explanation. No personal data is included.

Permissions

  • linkedin.com — Required to inject the art feed into your LinkedIn page.
  • storage — Required to save preferences and artwork cache locally.
  • Museum & Gemini API hosts — Required to fetch artwork data and AI explanations.

LinkedIn Data

The extension does not read, collect, or transmit any data from your LinkedIn account. It only hides the feed UI and displays artwork in its place.
